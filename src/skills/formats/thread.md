# X Thread (โพสต์ให้ความรู้แบบ Thread)

## ภาพรวม
เล่าเป็นลำดับความคิด แต่ละ "node" เป็นอิสระในตัวเอง
ไม่ใช่บทความที่ถูกตัดแบ่ง — แต่ละตอนต้องมี micro-payoff ของตัวเอง

## เป้าหมายหลัก
- ตอนที่ 1 = Hook ที่ทำให้อยากอ่านต่อ + บอกว่าจะได้อะไร
- แต่ละตอน forward เรื่องราวไปอีกขั้น ไม่ใช่แค่ filler
- จบแบบมี closure ไม่แขวนค้างไว้

## โครงสร้าง
```
[1/N] Hook + สัญญาว่าจะได้อะไรถ้าอ่านต่อ
[2/N] ประเด็นแรก
[3/N] ประเด็นที่สอง / ขยาย
...
[N/N] สรุป + hashtag 2-3 ตัว
```

## กฎ
- DO: ขึ้นต้นแต่ละ node ด้วย "1/", "2/" ฯลฯ
- DO: แต่ละ node ≤ 280 ตัวอักษร (X/Twitter limit)
- DO: ใส่ hashtag ท้าย node สุดท้ายเท่านั้น
- DO: node 1 ต้องอ่านแล้วรู้ว่า thread นี้เกี่ยวกับอะไร
- DON'T: ใช้ Markdown heading ใดๆ
- DON'T: ทำให้ node ใด node หนึ่งเป็นแค่ "bridge" ที่ไม่มีเนื้อหา
- DON'T: ทำ thread ยาวเกิน 10 nodes โดยไม่จำเป็น
