# บทความ Blog / SEO

## ภาพรวม
บทความยาวที่มีโครงสร้างชัด เน้น credibility และ information density
เหมาะกับเนื้อหาที่ต้องการบริบทลึก อธิบายขั้นตอน หรือเปรียบเทียบ

## เป้าหมายหลัก
- อ่านแล้วรู้เรื่องจริงๆ ไม่ใช่แค่ skimmable
- Heading ช่วยให้ navigate ได้ ไม่ใช่แค่ตกแต่ง
- จบด้วย ## บทสรุป เสมอ (ยกเว้น length = short)

## โครงสร้าง
```
[Intro]       — ระบุปัญหา / บริบท / ทำไมสำคัญ (ไม่ใช้ heading)
[## หัวข้อ 1] — แนวคิดแรก
[## หัวข้อ 2] — แนวคิดที่สอง
...
[## บทสรุป]   — สรุปประเด็นหลัก + takeaway
```

## กฎ
- DO: ใช้ ## Heading เมื่อเนื้อหา ≥ 400 คำ
- DO: แต่ละ heading = แนวคิดใหม่หนึ่งอย่าง ไม่ซ้ำกัน
- DO: ให้ข้อมูลแม่นยำ มีตัวเลข/ชื่อ/วันที่ถ้ามี
- DO: Intro ต้องระบุชัดว่าบทความนี้เกี่ยวกับอะไรและทำไมต้องอ่าน
- DON'T: ใส่ CTA ขาย (เพิ่มได้ถ้า custom instruction บอก)
- DON'T: Hype language — ใช้ภาษาที่วัดได้และเป็นกลาง
- DON'T: บทสรุปที่แค่ repeat เนื้อหาเดิมคำต่อคำ
